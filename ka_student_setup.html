Before You Start

Environment setup — your track page has track-specific git commands
Install Python 3.12, git, and VS Code ▼ 
# Install Homebrew if needed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install Python 3.12, git, VS Code
brew install python@3.12 git
brew install --cask visual-studio-code

# Verify
python3 --version   # should be 3.12.x
git --version
# In PowerShell (run as Administrator)
winget install Python.Python.3.12
winget install Git.Git
winget install Microsoft.VisualStudioCode

# Restart terminal, then verify
python --version
git --version

Getting Help

Stuck? Use AI as a tool, not a shortcut
Stuck? Take a screenshot and ask AI.

At any point during setup or track work, if you hit a wall:

  1. Take a screenshot of exactly what you see (error message, confusing output, broken page)
  2. Open ChatGPT, Claude, or Gemini
  3. Upload the screenshot and ask: "I'm a COGS 160 student working on [your track]. I see this. What should I do next to succeed at [your task]?"

This is not cheating — it's the methodology. AI-directed development means using AI as a tool, not pretending it doesn't exist. Your job is to verify the answer against your track's acceptance criteria.
