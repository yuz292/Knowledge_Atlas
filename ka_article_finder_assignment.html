Phase 1 — Tool Orientation & Manual Baseline

Before you can evaluate or improve the pipeline, you must experience the problem it solves. Spend Week 3 searching manually, then systematically learn each of the four AI search tools. Your Query Diary is the evidence base for everything that follows.

Week 3 task: Execute 15 manually-crafted queries on PubMed, PsycINFO, or Web of Science using Boolean operators. Record every query in the diary below. Identify 8 potentially relevant articles by title and abstract. Sample all four tools — Elicit, Google AI Scholar, Consensus, and Scholar AI — with at least 2 queries each. Write a 500-word reflection by the end of the week comparing natural language search against Boolean strings.
The Four Search Tools
🎓
Google AI Scholar
Natural Language · Free

Ask full English questions rather than Boolean strings. Best for exploratory queries where you do not know the controlled vocabulary. Returns summaries with citations, not just a list.

  • Ask questions, not keyword strings
  • Follow-up question chains work well
  • Cite a paper and ask "what else is related?"
  • Cannot filter by date or database
No API — manual only
⚗️
Elicit
AI Research Tool · API

Ask a research question; Elicit retrieves relevant papers and extracts structured fields (methodology, sample size, effect size, limitations). The easiest manual DL path for library-accessible PDFs.

  • Structured extraction per paper
  • API available — automation possible
  • Direct library DL is the fastest manual PDF method
  • Good for effect-size comparison across studies
API available — explore automation
🤝
Consensus
AI Research RAG · Chat

Available via Claude and Gemini chat. Specialises in synthesising consensus across studies — "does X cause Y?" — and returning a confidence rating with supporting citations.

  • Best for contested causal claims
  • Returns yes/no/mixed with citation support
  • Access via Claude chat or Gemini
  • API status uncertain — check during Week 3
API status: investigate
📖
Scholar AI
GPT Plugin · Chat

Available as a ChatGPT/Claude plugin. Can retrieve and read PDFs directly, which makes it useful for full-text queries rather than just abstract-level search.

  • Full-text access for open-access PDFs
  • Good for "what does this paper actually say?"
  • Access via Claude chat (plugin mode)
  • Less structured output than Elicit
Plugin — no standalone API
Critical difference: Boolean search requires you to know the controlled vocabulary in advance ("Kaplan + Attention + Restoration + Workplace"). Natural language search (Elicit, Google AI Scholar) lets you ask "do natural views from office windows reduce mental fatigue?" and allows the system to map your intent to the literature. Learning to write good natural language questions is a skill — use the Question Maker to practise.

Phase 2 — Question Making & Query Strategy

The hardest problem in systematic review is knowing what to ask. This phase develops your question-making skills and connects them to KA's internal knowledge about where the evidence is thin. Use the Question Maker tool to generate, test, and refine queries before executing them at scale in Phase 4.

Core principle: A good natural language question for an AI RAG has a clear dependent variable, a specified population or context, and implies a comparison. "How does morning light exposure affect reaction time in office workers compared to windowless workspaces?" outperforms "light cognition workplace." The Question Maker shows you templates for each tool.
Three sources of good questions
1

Paper clusters — what does KA already know?

KA's hidden paper_clusters identify recurring topics across the 263+ ingested papers. A cluster with 10 articles is an area of existing coverage; a cluster with 3 articles and a high VOI score is a gap worth targeting. Open the Question Maker to see the current cluster map and VOI heat.

→ Question Maker → Topic Browser
2

VOI locator — where should papers exist but don't?

KA's Value of Information scores identify where filling a knowledge gap would most change the system's credences. A high-VOI gap means many downstream beliefs depend on a claim that has thin evidence. These are the highest-priority search targets.

→ Gap Explorer / Question Maker → VOI Panel
3

Interpreter & argumentation layer — what is contested?

KA's argumentation layer tracks claims where existing evidence is split, or where rebutting evidence undercuts a warrant. These contested claims are exactly where new articles can shift the network most. Ask KA "what is the most disputed claim in [topic]?" and use the answer as a search question.

→ Ask KA → "What is most contested about X?"
Best Practices by Tool
🎓
Google AI Scholar
Best practices
  • Use a full research question, not keywords: "What is the effect of biophilic office design on self-reported cognitive fatigue?"
  • Specify the population: "in office workers" or "in university students" avoids irrelevant clinical literature
  • Seed with a known paper: "Papers similar to Kaplan 1995 on attention restoration in natural settings"
  • Follow the citation chain: "What papers cite [result]?" and "What does [result] cite?"
  • Ask for disagreement: "Are there studies that failed to replicate [finding]?"
⚗️
Elicit
Best practices
  • State the research question as a hypothesis: "Does exposure to natural light during work hours improve sustained attention performance?"
  • Use column extraction: ask Elicit to pull effect sizes, sample sizes, and study designs into a table for comparison
  • Filter by year: restrict to 1995–present to match KA's inclusion criteria
  • Use the library DL link: fastest manual PDF acquisition path
  • API: Elicit's API accepts a research question and returns structured results — explore batch automation
🤝
Consensus
Best practices
  • Ask a binary causal question: "Does [X] cause [Y]?" format works best
  • Use it for contested claims: Consensus is strongest when the literature is genuinely split
  • Access via Claude chat: paste your question and add "use Consensus to find relevant studies"
  • Follow up: "What studies found the opposite result?"
  • Do not trust the confidence rating alone: read the citations and verify
🌐
PubMed / PsycINFO
Best practices (Boolean)
  • Use MeSH terms: "Attention"[MeSH] AND "Work Environment"[MeSH] is more precise than free text
  • Combine with title/abstract field: [Title/Abstract] limits to high-relevance hits
  • Date filter: add AND ("1995"[PDAT] : "3000"[PDAT]) for KA-compatible range
  • DOI export: export results as CSV with DOIs for batch submission to Article Propose
  • Cross-database: the same search in PubMed vs. PsycINFO often returns non-overlapping results — run both

Zotero Integration Note

Zotero can theoretically automate PDF retrieval from library subscriptions, but in practice its success rate is low. Recommended workflow for PDF acquisition in order of reliability:

  1. Elicit direct library DL link — fastest manual method; click the library link on any Elicit result card
  2. DOI.org → Unpaywall — for open access versions of paywalled papers
  3. Google Scholar "All versions" — often includes a free PDF link
  4. Zotero browser connector — works well on publisher pages; use as a fallback
  5. Email the author — always works; turnaround is usually fast for CNFA-adjacent researchers
Phase 4 — Scale-Up Campaign
150+
New citation records by end of Week 7. This means APA citations with DOIs logged in the system. PDFs should be acquired for every article where the abstract passes the relevance filter (target: 50+ PDFs), but citation-level records count toward the primary milestone. The question set you built in Phase 2 is your input. The Article Propose intake pipeline is your output channel. Exceptional teams should aim beyond 150 — the pipeline has no ceiling.

Campaign Pipeline
1

Generate questions from KA's knowledge state

Pull the top 20 highest-VOI topic areas from the Gap Explorer. For each, use the Question Maker to generate 3–5 tool-optimised questions. Focus on clusters with fewer than 20 articles and VOI ≥ 0.6.

Tools: Question Maker → VOI panel → export question set
2

Run questions via Elicit API (automated path)

Elicit's API accepts a research question and returns structured results including DOIs, titles, abstracts, and methodology fields. Batch 50 questions through the API, parse the JSON output, and extract a citation list. Document your API call pattern for the handoff package.

Tools: Elicit API (explore key in Week 4), Python batch script, CSV output
3

Run questions manually via Google AI Scholar

For questions where the Elicit API yields fewer than 5 results, run the question manually in Google AI Scholar. Copy the citations into the shared queue. Do not skip this — Google AI Scholar accesses different corpora and finds papers Elicit misses.

Tools: Google AI Scholar chat, shared citation queue in Question Maker
4

Screen citations for relevance (abstract level)

For every retrieved citation, record DOI/title in your corpus table or queue and read the abstract before spending effort on PDF acquisition. Decide: Include / Exclude / Uncertain. Only Include-status articles advance to PDF acquisition. Track 2 must preserve citation structure here, not just collect PDFs later, and should capture the abstract text whenever the search tool exposes it.

Tools: Article Search / corpus table / question queue → abstract-level relevance decision
5

Acquire PDFs for Include-status articles, then stage them with citations

Use the Elicit library link first. Fallback order: Unpaywall → Google Scholar free version → Zotero connector → author email. Once an article passes abstract screening and you have the PDF, upload it through Article Propose together with a usable citation record and the abstract if you have it. For Track 2, a bare PDF is incomplete.

Tools: Elicit DL link, Unpaywall, Article Propose → citation-plus-PDF staging with duplicate check
6

New area: Neuroscience of attention pathways

One explicitly requested expansion area is deepening KA's neuroscience coverage — specifically the neural pathways underlying sustained attention, cognitive fatigue, and stress recovery (prefrontal cortex, locus coeruleus-norepinephrine, default mode network suppression). Add 10–15 focused questions on these pathways to the question queue.

Seed questions: "LC-NE modulation of sustained attention in fatigued states" · "DMN deactivation during focused work" · "PFC executive function and environmental noise"
Neural Pathway Assignment Front

For some Track 2 teams, the assignment is not just "find more papers." It is to deepen one pathway family for the future Neural Underpinnings Explorer. That means collecting classic anchors, reviews, direct-measurement papers, and environment-relevant papers for a specific pathway.

Phase 5 — Pipeline Report & Handoff

Your final deliverable is a short technical report and a handoff package that makes it possible for fall-semester students to operate the article-finding pipeline independently. The report combines your recall/precision analysis with your campaign yield and lessons learned.

Handoff package must include: (1) query bank with at least 50 tested questions annotated by tool and topic cluster; (2) API call pattern for Elicit with sample code; (3) step-by-step PDF acquisition decision tree; (4) recall/precision analysis against your Phase 3 corpus; (5) list of 5 specific improvements to the pipeline with implementation notes.
Efficient citation capture: when a search tool shows a results page, copy the visible citation block or results text into a scratch document, delete the rows you do not want, then paste the remaining lines into Article Propose. This is faster and safer than trying to reconstruct citations from orphan PDFs later. Include first author, title, year, DOI, and abstract whenever the tool exposes them.

How many articles do I need by Week 8?
Target: 150 citation records with 50+ PDFs acquired. You should start early (Week 3) to give yourself enough time for the pipeline to mature. Aim for 30+ citations per week starting Week 3.
What counts as a valid source?
Peer-reviewed journals, published conference proceedings, or pre-prints from arXiv/bioRxiv with significant citation counts (≥5 cites). Avoid theses, dissertations, and grey literature unless explicitly instructed.
What if I can't find the PDF?
Use Unpaywall first, then the UCSD library proxy search, then request via ResearchGate or author email. If all fail, flag it in your log as "PDF-unavailable" and document the query chain for that article.
Which search tool should I use?
Start with Elicit (best for natural language), sample Google AI Scholar and Consensus to compare results. By Week 4, converge on the tool that best matches your search goals. Phase 5 report should justify your tool choices.
