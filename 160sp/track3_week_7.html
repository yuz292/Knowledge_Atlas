Week 7

Question for the week: How do you turn your technical pieces into a workflow a researcher could actually use?

Use AI well

Before you ask AI for help, make sure you can state what you are trying to learn this week and what evidence would count as a useful answer.

Class meetings

Meeting A — Build the researcher workflow
Start here: Sketch the shortest path from selecting a room to receiving a usable variant package.

In class

  • Sketch the smallest useful interface or workflow.
  • Decide what the user chooses, what the system chooses, what gets previewed, and what gets exported.
  • Keep the workflow narrow: pick a room, choose factors or target variables, generate a variant, inspect the log, export the package.
  • Use AI to suggest interaction patterns only after you know what information must appear on screen or in the package.

Home

  • Build a lightweight prototype of the workflow, even if some parts are mocked by structured placeholders.
  • Prepare one end-to-end path a researcher could follow without learning your internal implementation details.
  • Document where the workflow still depends on an expert operator.
Meeting B — Rehearse the final bundle
Start here: Pretend another lab receives your package next week. What must be obvious immediately?

In class

  • Create a complete experiment-style bundle: base room, variants, change log, limitations note, and quick-start instructions.
  • Ask another team to use the bundle without verbal help.
  • Record the points at which they become confused or blocked.
  • Revise the workflow and package based on that test.

Home

  • Finalize the main artifacts for the final integration meeting.
  • Prepare a three-minute explanation of what your system supports, how it works, and what remains unsolved.

Week deliverables

D10 Researcher workflow prototype

Rationale
The studio's value lies in making transformation usable for someone who is not already a VR expert.

What to submit
A prototype workflow or interface plus one experiment-style bundle that a new user can inspect and understand.

What good work looks like
The path is short and comprehensible. The user can see what choices were available and what changes were made. The prototype does not hide limitations.

Example
A minimal web or slide-based workflow that lets a user pick a benchmark room, select target factors, inspect the resulting log, and download the variant package.
