PDF Relevance Filter

Given a K-ATLAS question and a submitted paper, determine whether the paper provides evidence relevant to that question. This is the foundation of quality control for the entire corpus.

THE PROBLEM

Knowledge Atlas has a question bank of 130 research questions spanning 34 topics — from Attention Restoration Theory to Fractal Patterns. Students and automated pipelines submit PDFs as evidence for these questions. But how do you know a paper is actually relevant? A paper about “attention deficit hyperactivity disorder in natural classroom settings” mentions both “attention” and “natural,” but it is about clinical ADHD, not attention restoration from nature exposure. A paper about forest bathing and cortisol is adjacent to an attention restoration question — same environment, wrong outcome measure. If irrelevant papers enter the corpus, every downstream analysis is contaminated.

We need a filter that takes a K-ATLAS question and a submitted PDF and determines: does this paper provide evidence relevant to this specific question? The filter must understand what “relevant” means in the context of each question’s evidence type, population, and environmental variable. This is the foundation of quality control for the entire system.

STEP 2 Write Spec
COMPONENT: PDFRelevanceFilter INPUT: question (from question_bank, 130 questions across 34 topics) paper_title (string), paper_abstract (string) paper_type (optional enum), paper_year (optional int) OUTPUT: Verdict (ACCEPT / REJECT / MANUAL_REVIEW) with per-check reasoning DATA SOURCE: question_bank.json (130 questions with topic, search_hints, difficulty) articles.json (760 papers with article_type distribution) Topic keyword banks (env-side + outcome-side + false-keyword exclusions) PIPELINE (7 sequential checks, stops on hard failure): CHECK 1 — Document Type Acceptability Accept: empirical_research, systematic_review, meta_analysis, narrative_review, case_study, mixed_methods, qualitative Marginal (warn): theoretical, conference_paper, thesis, book_chapter Reject: editorial, book_review, preprint, computational_simulation Auto-detect from abstract if not specified CHECK 2 — False Keyword Detection Each topic has an exclusion list of terms that LOOK relevant but indicate a different domain: - ART: "attention deficit", "ADHD", "attention disorder" - SRT: "heat stress", "material stress", "structural stress" - Lighting: "light rail", "light microscopy" Paper containing ANY exclusion term → REJECT with explanation CHECK 3 — Topic Overlap (Both Sides) Each question maps to environment keywords AND outcome keywords. Paper must match at LEAST ONE from EACH side. - Both matched → PASS - One side only → WARN (partial relevance) - Neither → FAIL CHECK 4 — Evidence Type Match Detect empirical markers (statistics, sample sizes, methods) vs. review markers vs. theoretical markers in abstract. Empirical or review → PASS. Theoretical only → WARN. CHECK 5 — Dependent Variable Alignment Extract expected DVs from question text and search hints. Match against paper abstract. Key case: shinrin-yoku paper measures cortisol (stress DV) but question SQ-001 asks about attention (cognitive DV). Same environment, wrong outcome → WARN (not PASS). CHECK 6 — Relevance Tier Classification DIRECT: topic match + DV match + evidence match INDIRECT: topic match but wrong DV (adjacent) CONTEXTUAL: right DV but wrong environment IRRELEVANT: neither matches CHECK 7 — Final Reasoning Synthesize all checks into human-readable verdict. All pass → ACCEPT. Any fail → REJECT. Warnings → MANUAL_REVIEW. EDGE CASES: - Topic not in keyword bank → extract keywords from question text - Abstract too short to detect type → default to empirical, flag uncertainty - Paper relevant to DIFFERENT question → note which question it fits
STEP 3 Acceptance Tests
QUESTION: SQ-001 (Attention Restoration Theory, easy) "What empirical evidence supports the claim that natural environments restore directed attention better than urban environments?" TEST 1: Direct hit — ART experiment INPUT: "Attention restoration and performance enhancement in a simulated natural environment after cognitive fatigue" EXPECT: ACCEPT, Tier DIRECT, all 7 checks pass VERIFY: Environment keywords (nature, natural) AND outcome keywords (attention, fatigue, restoration) both match TEST 2: Off-topic — structural engineering INPUT: "Structural analysis of reinforced concrete beams under cyclic loading conditions" EXPECT: REJECT at Check 1 (document type: computational_simulation) VERIFY: Checks 2–7 show "Skipped (pipeline stopped)" TEST 3: Adjacent but wrong DV — shinrin-yoku/cortisol INPUT: "Shinrin-yoku (forest bathing) and salivary cortisol responses: A field experiment in temperate forests" EXPECT: MANUAL REVIEW, Tier INDIRECT VERIFY: Check 3 (topic overlap) passes — nature keywords match Check 5 (DV alignment) warns — cortisol ≠ attention This is the HARD case. The filter must not accept it as DIRECT evidence for an attention question. TEST 4: False keyword match — ADHD study INPUT: "Attention deficit hyperactivity disorder in natural and urban classroom settings: A comparative study" EXPECT: REJECT at Check 2 (false keyword: "attention deficit") VERIFY: The word "attention" appears but in clinical context. Filter catches "attention deficit" as exclusion term. TEST 5: Indirect methodological — ARS validation INPUT: "Measuring directed attention fatigue: Validation of the Attention Restoration Scale across three populations" EXPECT: MANUAL REVIEW, Tier INDIRECT or DIRECT VERIFY: Relevant to the question's theoretical framework but measures instrument validity, not the IV→DV effect.

What to Notice

The spec did most of the intellectual work. The 7 checks, the keyword banks, the tier classification rules, the false-keyword exclusions — all of that was designed before a single line of code was written. The AI wrote the JavaScript in minutes. But the spec is where the epistemology lives.

Test 3 (shinrin-yoku/cortisol) is the case that matters most. A naive keyword filter would accept it — “forest,” “nature,” “experiment” all match. But the question asks about attention, and this paper measures cortisol. Same environment, wrong dependent variable. The filter catches this at Check 5 (DV Alignment) and classifies the paper as INDIRECT rather than DIRECT. That distinction is the difference between a contaminated corpus and a clean one.

When something goes wrong during the demo, the fix is always in the spec — adding a missing keyword, adjusting a tier rule, refining a false-keyword exclusion — never in the rendering code.
