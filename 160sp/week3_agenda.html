Everyone · First 30 Minutes

Step 1: Git & GitHub Setup

Every student needs git working on their laptop and access to the project repos before doing anything else. If you did this in Weeks 1–2, verify your setup still works. If not, follow these steps exactly.

1

Install Git

Mac: Open Terminal. Type git --version. If it's not installed, macOS will prompt you to install Xcode Command Line Tools — say yes. Alternatively: brew install git.
Windows: Download from git-scm.com. Use the defaults during installation. Use Git Bash for all commands below.
Linux: sudo apt install git

2

Create a GitHub Account (if you don't have one)

Go to github.com/join. Use your UCSD email — this qualifies you for GitHub Education benefits (free Copilot, etc.).

One Required Course Step

After you create or verify your GitHub account, open the GitHub Access page and save your GitHub username. If your GitHub username is exactly the same as the part of your UCSD email before the @, you can confirm it with one click. If it is different, submit the actual username there.

3

Configure Git Identity

Open your terminal and run these two commands (replace with your info):

# Tell git who you are (used in commit messages) git config --global user.name "Your Name" git config --global user.email "you@ucsd.edu"
4

Clone the Knowledge Atlas Repo

This is the main repo where the website, course pages, and coordination files live. Everyone needs it.

# Navigate to where you want the project (e.g., a COGS160 folder) mkdir -p ~/COGS160 && cd ~/COGS160 # Clone the repo git clone https://github.com/dkirsh/Knowledge_Atlas.git # Enter the repo and verify cd Knowledge_Atlas git status

You should see: On branch master, nothing to commit, working tree clean.

5

Create Your Personal Branch

Never commit directly to master. Create a branch for your track work:

# Replace N with your track number (1, 2, 3, or 4) and your-name with your name git checkout -b track/N-staging/your-name # Push it to GitHub so it exists remotely git push -u origin track/N-staging/your-name

Example: git checkout -b track/2-staging/maria-chen

Verify Your Setup

Run git log --oneline -5. You should see recent commits. Run git branch — your branch name should be highlighted. If either fails, ask a teammate or TA for help before proceeding.

Understand the Database

Article Eater uses SQLite. The key table is articles with columns: article_id, title, abstract, doi, corpus_id, authors, year, venue, full_text, sections, citation_count, is_open_access, url_pdf. There is also a processing_queue for job management.

# Inspect the database schema cd ~/COGS160/Article_Eater python3 -c "import sqlite3; conn = sqlite3.connect('data/article_eater.db'); print([r[1] for r in conn.execute('PRAGMA table_info(articles)')])"

Understand the Site Architecture

The site has two navigation regimes: a global nav (Explore, Topic Map, Evidence, Gaps, Articles, Contribute) for researchers and contributors, and a COGS 160 nav (Syllabus, A0, A1, Tracks 1–4) for course students. There are also user-type-specific home pages (student, researcher, contributor, instructor, practitioner). Your job is to make each user's journey through the site clear and effective.
