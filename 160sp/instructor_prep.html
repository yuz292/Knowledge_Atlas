Do now Before Day 1 (Tuesday Week 1)

KA site is live and registration works

Test the full flow: ka_register.html → ka_login.html → ka_user_home.html. Confirm the auth server is running and students can create accounts. Test on a fresh browser with no cached state.

GitHub repo is public and cloneable

Verify git clone https://github.com/dkirsh/Knowledge_Atlas.git works from a machine that has never accessed the repo. Students need this by Week 3 but you should confirm it now.

Question bank has enough questions for all students

Each student gets 2 questions (A0 assignment). With N students you need at least 2N unique question assignments. The QUERY_BANK has 130 questions across 11 topics. Prepare the assignment sheet mapping student → questions.

AI subscription recommendation is accurate

Confirm pricing for Claude Desktop ($20/mo), ChatGPT Plus ($20/mo), and alternatives. Verify that free tiers are genuinely insufficient for sustained multi-turn sessions so the recommendation is honest.

Week 1 slides or talking points ready

You need material for: syllabus overview, KA system walkthrough (use ka_explain_system.html live), 7 AI failure modes presentation, and A0 assignment introduction. The failure modes content is in the Designing_Experiments course site and also embedded in the Week 1 agenda page.

A0 assignment page is complete and linked

Verify ka_collect_articles.html renders correctly, links to the Google search guide, and the contribute page accepts article submissions. Test the upload flow end to end.

All weekly agenda pages are live

Eight agenda pages (week1–8), the track signup page, and the schedule page with "View Full Agenda" links. Navigate the full chain: Week 1 → 2 → 3 → … → 8 using the next buttons. Verify no broken links.

Push all local commits to GitHub

You are currently 8+ commits ahead of origin. Run git push origin master from your Mac terminal. Students cannot see the new pages until they are on GitHub.

Week 2 prep Before Week 2 (AI Demo Day)

Demo task prepared: PDF Relevance Filter

The Week 2 demo requires a pre-loaded first prompt (role setup + context + spec). Write and test this prompt in advance. Have 3–5 labeled example PDFs ready (known relevant and known irrelevant) for the verification step. The other prompts are named but intentionally unscripted — you will improvise based on the AI's output.

Student task bank populated

Students begin their own AI tasks in Week 2. The tasks should be drawn from ka_thursday_tasks.html or equivalent. Each task needs a corresponding context file (in 160sp/context/). Verify that context files exist for all four tracks.

Submission mechanism for methodology logs

Students must submit their methodology log and task output at the end of Week 2. Decide: Google Form? Canvas? Git commit? Email? Whatever you choose, test it and include the link on the Week 2 agenda page.

Week 3 prep Before Week 3 (Track Formation)

Track signup page is functional

Verify ka_track_signup.html renders correctly and the signup form submits data somewhere you can read it. Students finalize track choice this week.

GitHub branch structure ready for teams

Create the four staging branches: track/1-staging, track/2-staging, track/3-staging, track/4-staging. Students will create personal branches off these. Set branch protection rules if desired.

Sprint 1 contract template available

Students write Sprint 1 contracts on Thursday. Provide a template or example contract. The format is described in the Week 3 agenda but a downloadable template saves time.

Track assignment pages reviewed for clarity

Each track assignment page (ka_tag_assignment, ka_article_finder_assignment, ka_vr_assignment, ka_gui_assignment) must be self-explanatory — reading it alone should be enough for a student to understand their Sprint 1 deliverable. Read them fresh, as if you were a student, and fix any ambiguity.

Ongoing throughout the quarter

Keep the methodology log submission system working. Review and approve Sprint contracts within 48 hours of submission. Monitor the article corpus for quality — spot-check 10% of student-submitted PDFs for relevance. Update weekly agenda pages if plans change.
