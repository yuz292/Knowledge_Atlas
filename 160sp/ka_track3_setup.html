Track 3 Setup — AI & VR Room Builder

Install Unreal Engine 5, the class's room-template starter project, and the AI room-modification harness. This is the heaviest setup of the four tracks — plan for an hour on a fast machine, longer if Unreal needs to compile shaders.

Prerequisites

  • Windows 10/11 or macOS 13+. Linux is technically supported but unsupported for the class. Ask a TA if you need a loaner machine.
  • Discrete GPU with at least 6 GB VRAM (RTX 2060 / M1 Pro / equivalent). Intel integrated graphics will not render the template scenes at an interactive frame rate.
  • 80 GB free disk. Unreal plus the starter content is large.
  • Epic Games account — free; sign up at epicgames.com.
  • Python 3.11+, Git as in the other tracks.
  • Quest 2/3, Valve Index, or equivalent headset if you are on the VR-authoring sub-team. Not required for the AI-harness sub-team.
  • Blender 4.x (free) — used as a neutral staging ground for meshes that come out of text-to-3D services before they are imported into Unreal. Download from blender.org. Choose the LTS build if you prefer stability over the newest features.
  • A text-to-3D account — either Meshy or Tripo3D. Both services take a prompt (or an image) and return a textured mesh in a few minutes. Free tiers are enough for classwork; pick one and stay with it so your asset provenance is consistent.

Step-by-Step

1Install Epic Games Launcher & Unreal Engine 5.4

Download the Epic Games Launcher, sign in, and install Unreal Engine 5.4 from the "Unreal Engine" tab. Accept the default components; the install is 30–50 GB.

2Clone the KA-Rooms starter project
git clone git@github.com:kirsh-lab/KA-Rooms.git
cd KA-Rooms
git lfs pull   # pulls the base-room .uasset files
Git LFS is required — install once with brew install git-lfs (macOS) or from git-lfs.com on Windows, then run git lfs install.
3Open the project and let Unreal compile shaders

Double-click KA_Rooms.uproject. The first open triggers a shader compile that can take 15–40 minutes. Let it finish before touching anything.

4Install the AI room-modification harness
cd ai_harness
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# Edit .env: ANTHROPIC_API_KEY=...
5Run the harness against a base room
python -m ka_rooms.ai_harness \
    --room living_room_v1 \
    --instruction "raise the ceiling to 3.6 m and curve the back wall"

The harness writes a modified .uasset into KA-Rooms/Content/Generated/; reload the Unreal editor to see the result.

6Set up your asset-generation pipeline

Most of the Week 3–5 rooms will need one or two custom props that the Unreal Marketplace does not cover (a particular chair shape, a specific artefact on a shelf, a culturally distinctive light fixture). The class uses a three-step pipeline:

  1. Generate a textured mesh in Meshy or Tripo3D from a short prompt ("a round wooden side-table, 45 cm tall, mid-century style") or a reference image. Export as .glb or .fbx.
  2. Clean up in Blender: check scale against a reference cube, delete stray geometry, consolidate materials, and bake down texture atlases if the mesh arrived with several overlapping UV islands. Re-export as .fbx.
  3. Import into Unreal through the Content Drawer → Import button and drop into Content/Generated/props/. Check collision and lightmap UVs before saving.
Why Blender sits in the middle. Meshes that come out of text-to-3D services are rarely clean enough to import straight into Unreal — scales are off by a factor of ten, UV seams are visible, and materials are sometimes duplicated. Blender is the neutral editor where you correct these issues once, so the Unreal import is predictable. Skipping this step is the most common reason a generated prop looks "wrong" in the headset.
7(VR sub-team only) Pair your headset

Quest users: install Meta Quest Link, enable developer mode on the headset, and follow Epic's VR preview quick-start. Index users: use SteamVR with OpenXR runtime set to SteamVR.

Common Snags

  • Unreal crashes on first open — usually GPU driver. Update to the latest NVIDIA or AMD drivers and retry.
  • LFS files missing — re-run git lfs install then git lfs pull. If the repo has old clone metadata, delete the checkout and clone afresh.
  • Harness writes succeed but Unreal shows nothing — Unreal caches assets. Right-click the Generated folder in the Content Browser and choose Refresh.
Branching rule. Create track3/<your-handle>/<room-name> for every new room. Large binary commits go through Git LFS only.
