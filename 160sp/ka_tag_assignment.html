Claim a Computational Tag Assignment

Each assignment bundle defines five image-analysis computations you will implement as code, run over a batch of workspace images, and submit for review. Select one or more bundles, then click Submit Claim. You will receive a confirmation email and the assignment will appear in your contributor profile.

▸ STEP 0: Environment Setup (expand if this is your first time)

1. Prerequisites

You'll need these tools installed:

Python 3.12 · git · VS Code

macOS:

brew install python@3.12 git && brew install --cask visual-studio-code

Windows:

winget install Python.Python.3.12 Git.Git Microsoft.VisualStudioCode

2. Clone & Create Your Branch

Run these commands in your terminal:

git clone https://github.com/dkirsh/Knowledge_Atlas.git cd Knowledge_Atlas git checkout track/1-staging git checkout -b track/1-staging/your-ucsd-username git push -u origin track/1-staging/your-ucsd-username

Replace your-ucsd-username with your UCSD username (e.g., jdoe).

3. Student Database

Important: The student database copy is at data/student/web_persistence_v5_student_copy.db
NEVER write to data/web_persistence_v5.db (the production database).
If you accidentally do: STOP and message the instructor immediately.

4. Submit Your Work

When you're ready:

  1. Commit your changes: git commit -m "[T1 Week X] Brief description"
  2. Push to your branch: git push
  3. Open a Pull Request with base: track/1-staging, compare: your branch

Stuck? Take a screenshot and ask AI.
At any point, if you hit a wall:
How to get help from ChatGPT, Claude, or Gemini:
  1. Take a screenshot of what you see
  2. Open ChatGPT, Claude, or Gemini
  3. Say: "I'm a COGS 160 student on Track 1 (Image Tagging). I see this screenshot. What should I do next to succeed at claiming my tag bundles?"

This is the methodology — AI as a directed instrument, not a chatbot.

How do I know if my tags are correct?
Check against the Phase 0 reference table and aim for inter-rater agreement ≥ 0.75 with a partner. Ambiguous cases should be documented in your rationale.
What if an image doesn't fit any bundle?
Flag it in your rationale document. Ambiguous images are valuable data, not errors — they help us understand edge cases in the protocol.
Where do I submit?
Commit to your branch, push, and open a Pull Request to track/1-staging. See STEP 0 above for details.
What if I make a mistake on a submitted assignment?
Open a new commit on your branch with the fixes. Instructors will review the latest commit; you can iterate until sign-off.
