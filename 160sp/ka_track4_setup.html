Track 4 Setup — Tuesday of Week 3

Interaction Design is primarily a design track, but you still need the site and backend running on your own machine so that the pages you critique and the journeys you map are the ones the class is actually shipping. Plan for 25–40 minutes on Tuesday of Week 3.

Prerequisites

  • Python 3.11+, Git, a GitHub account with SSH keys — same as the other tracks.
  • A modern browser with DevTools (Chrome, Firefox, or Safari with the developer menu enabled).
  • Figma account (free) for mid-fidelity mock-ups and journey maps.
  • Optional but recommended: the Axe DevTools browser extension for quick accessibility checks on pages you review.

Install the Shell Tools (by OS)

macOS (Homebrew)

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install python@3.12 git git-lfs
git lfs install

Windows (winget)

winget install Git.Git
winget install Python.Python.3.12
winget install GitHub.GitLFS
git lfs install

Windows users: open PowerShell as Administrator the first time. If winget is missing, install the App Installer from the Microsoft Store.

Step-by-Step

1Clone the site repo
git clone git@github.com:kirsh-lab/Knowledge_Atlas.git
cd Knowledge_Atlas
2Create a virtual environment
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\Activate.ps1
pip install --upgrade pip
pip install -r requirements.txt
3Boot the local server
python 160sp/ka_local_server_setup.py
# Site now live at http://localhost:8765

Open a browser at http://localhost:8765. You should see the K-ATLAS home page. Navigate to Track 4 and confirm this page renders.

4Receive your persona assignment

Your TA will assign you one of the eight personas (P1–P8, see Track 4 §2). Paste the assignment into your 160sp/personal/<your-handle>.md file, commit to a branch named track4/<your-handle>/persona-assignment, and open a PR.

5Pre-read before Thursday
  • Cooper (1999/2014), The Inmates Are Running the Asylum, Ch. 1.
  • Flanagan (1954), The critical incident technique.
  • Beyer & Holtzblatt (1997), Contextual Design, Ch. 1 (skim).

Common Snags

  • Port 8765 in use — another student probably left a server running. Kill it with lsof -ti:8765 | xargs kill (macOS/Linux) or use --port 8766.
  • Mixed CSS on refresh — hard-reload with Cmd/Ctrl+Shift+R; the shell CSS is aggressively cached.
  • Git pushes rejected — you probably tried to push to main. Create a track4/<handle>/… branch and push that.
Branching rule. Never commit directly to main. Every page review, every journey map, every critique goes on a branch named track4/<your-github-handle>/<short-topic> and comes in through a PR.
