# T1.f — Published tag set with provenance (500 images)

**Track**: T1 (Image Tagging)
**Span**: Weeks 6–7
**Hardness**: Hard
**Points**: 15
**Verification**: Automated (DB count + provenance audit) + AI re-grade of provenance quality

---

## What the student must produce

1. **500 images** in the Atlas tag DB with finalised tags, each row carrying: `image_id`, `final_tag`, `final_confidence`, `provenance.taggers` (student_id + any co-tagger), `provenance.kappa_if_paired`, `provenance.hitl_adjudicated` (boolean), `provenance.classifier_score` (if T1.e model was used as assist).
2. A **provenance-quality report** (markdown, 400–500 words) describing the mix of tagging sources across the 500, the proportion that went through paired kappa review (T1.c) or HITL adjudication (T1.d) or classifier-assist (T1.e), and the error-rate estimate for the published set (derived from any disagreement the student can compute).
3. A **release manifest** — a CSV or JSON file — that downstream users of the tag set can consume.

## Scoring bands

### Completeness

- **0** — < 200 published rows.
- **1** — 200–399.
- **2** — 400–499 rows, provenance field populated, release manifest present, provenance report meets 350-word minimum.
- **3** — 500 rows, provenance populated, release manifest + README, provenance report 450+ words.

### Quality

- **0** — Rows published but provenance empty or nonsensical; release manifest broken.
- **1** — Provenance present but thin (most rows carry only `taggers` with no kappa/HITL/classifier source).
- **2** — ≥ 30 % of rows carry kappa or HITL provenance; release manifest consumable by a second student.
- **3** — ≥ 50 % with kappa/HITL provenance; release README includes an error-rate estimate with a defensible derivation.

### Reflection

Standard bands. Prompt: "If someone else picks up this 500-image tag set as training data, what should they know about its limitations?"
