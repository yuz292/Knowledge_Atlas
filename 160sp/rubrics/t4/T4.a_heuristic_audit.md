# T4.a — Heuristic audit of K-Atlas public site (10 findings)

**Track**: T4
**Span**: Weeks 3–4
**Hardness**: Medium
**Points**: 12
**Verification**: Automated (findings-file schema check) + AI re-grade of finding quality

---

## What the student must produce

Ten structured findings against the K-Atlas public site, applying Nielsen's 10 usability heuristics (Nielsen, 1994). Each finding is one markdown file at `160sp/tracks/t4/{student_id}/findings/T4.a_{nn}.md` with a fixed schema.

## Finding-file schema (every T4 finding uses this)

```markdown
# T4.a-{nn} — {short title}

**Heuristic**: {one of Nielsen's 10}
**Scenario**: {one-line scenario the user was attempting}
**Page**: {URL}
**Observed behaviour**: {literal description, 30–80 words}
**Expected behaviour**: {what would make the scenario succeed, 20–50 words}
**Impact**: {who is affected and how, 20–40 words}
**Severity**: {1–5 per Nielsen: 1=cosmetic, 5=catastrophic}
**Screenshot**: {path to screenshot file in ./screenshots/}
**Recommendation**: {one-paragraph suggested fix, 40–80 words}
```
