The Demo Task: PDF Relevance Filter

Given a K-ATLAS question and a submitted PDF, determine whether the paper provides evidence relevant to that question. This is the foundation of quality control for the entire corpus — if irrelevant papers enter the system, every downstream analysis is contaminated. The task requires understanding what "relevant" means in the context of each specific question's evidence type, population, and environmental variable.

Prompts Prepared for the Demo
Prompt 1 (Ready): Role setup + context loading + spec definition. "You are a bibliometrician who specializes in systematic review methodology..."
Prompt 2 (Named): Implementation — build the filter against the spec
Prompt 3 (Named): Verification — test against labeled examples
Prompt 4 (Named): Edge cases — papers that are adjacent but not relevant

Prompts 2–4 are named but not pre-written. What David prompts depends on what the AI produces. This is how real AI-directed development works — you cannot script the whole session.

Your goal: complete step 1 (role setup and context loading) and step 2 (specification) of the seven-step workflow before class ends.

Why Context Files Matter

An AI without context is an AI that hallucinates. The context file gives your AI the vocabulary, the data structures, the acceptance criteria, and the constraints it needs to produce work that actually fits the system. Without it, the AI will produce plausible-looking output that uses the wrong field names, misunderstands the data model, and fails every integration test.
