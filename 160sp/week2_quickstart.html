Exercise Quickstart Guide

Everything you need to start your exercise — in 5 minutes.

Use this as a helper page. The main assignment brief is on Exercises. The live data pages are optional reference pages if you want to inspect the corpus before building.

What You Need

These exercises require no software installation, no repository cloning, and no terminal commands. You work entirely through an AI chat interface (Claude, ChatGPT, or similar) and a web browser.

You need exactly three things:

  1. An AI assistant — Claude (claude.ai), ChatGPT, or another LLM chat
  2. Your context file — a markdown document you download and drag into your AI chat (see table below)
  3. A way to view HTML — your browser, or the AI's built-in preview if available
No git clone needed. The data files are hosted on the web and your AI-generated code will load them directly from the server.

  • Drag and drop the downloaded .md file directly into the chat window
  • Use the attachment button (📎) to upload the file
  • Or if those don't work: open the file, select all, copy, and paste into the chat

Then type this as your first message:

"I've attached the context file for my exercise.
Please read it carefully and explain:
1. What is this task about?
2. What are the key data structures I'll work with?
3. What's the hardest part going to be?"

If the AI's explanation doesn't match the context file, correct it before proceeding. An AI that misunderstands the data model will produce code that looks right but crashes on real data.

3

Follow the 7-step method from the exercise page. The context file has already primed your AI. Now proceed through: Panel → Specification → Tests → Implementation → Verification → Diagnosis → Fix.

Running Your Code

Your AI will generate a standalone HTML file for your component. There are three ways to handle the data — pick whichever works for you:

Option A: Embedded Sample Data (Recommended — Works Everywhere)

Ask the AI to generate 20–30 representative data records inline in the HTML file, based on the schema in your context file. The AI already knows the data structures, so it can create realistic sample data. Your code will look like:

const SAMPLE_DATA = [
  { "belief_id": "PDF-0844_MPX_007", "content_preview": "Viewing natural landscapes...", ... },
  { "belief_id": "PDF-0331_MPX_003", "content_preview": "Green spaces reduce cortisol...", ... },
  // ... 20-30 more records
];

This works in AI previews (Claude artifacts, ChatGPT canvas), in any browser, and needs no server. It's the fastest path to a working component.

This is the recommended approach. The grading rubric evaluates your methodology (panel, spec, tests, reflection), not whether you processed all 1,900 records. A working component on 25 representative records is a complete submission.

Option B: AI Preview with Server Fetch

If your AI has a built-in code preview (Claude's artifacts, ChatGPT's canvas), you can try loading the full dataset from the server:

const resp = await fetch('https://xrlab.ucsd.edu/ka/data/ka_payloads/evidence.json');
const data = await resp.json();

This may or may not work depending on your AI tool's sandbox environment. If it fails with a "CORS" or "network" error, switch to Option A — it's not worth debugging.

Option C: Local Files + Local Server

If you want to test against the full 1,900-record dataset on your own computer:

  1. Create a folder for your work (e.g., week2/)
  2. Save the AI's HTML output as my_exercise.html in that folder
  3. Create data/ka_payloads/ inside your folder and download:
    ↓ evidence.json (5.5 MB) ↓ argumentation.json (4.5 MB)
  4. Open a terminal in your week2/ folder and run: 
    python3 -m http.server 8000
  5. Open http://localhost:8000/my_exercise.html in your browser

This gives you the full dataset locally. Press Ctrl+C to stop the server when done.

Common Mistakes

  • Don't skip the context file. Without it, your AI will hallucinate data structures and field names. Everything downstream fails.
  • Don't skip Steps 1–3. Jumping straight to "write me the code" produces plausible-looking components that fail on real data.
  • Don't get stuck on data loading. If your code can't fetch from the server, use embedded sample data (Option A above). Don't spend exercise time debugging network issues.
  • Don't worry about the final code being perfect. Grading is on methodology (panel quality, spec completeness, verification rigor, reflection honesty) — not on whether your component is bug-free.

Pre-Submission Checklist

  • Context file was pasted as the first AI message
  • Panel transcript with real disagreement between panelists
  • Specification with inputs, outputs, edge cases, success conditions
  • At least 4 acceptance tests with expected outcomes
  • All AI prompts included (including failed attempts)
  • Verification results for each test (pass/fail with evidence)
  • Reflection paragraph 1: Surprise from the panel discussion
  • Reflection paragraph 2: Specific failure + how you diagnosed it
