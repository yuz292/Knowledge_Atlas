Track 3 Contract

Use this rule: Track 3 students may inspect and edit assets in Blender, but the final course submission path is browser-runnable A-Frame / WebXR. Unity and Unreal are optional prototype environments, not the required student baseline.

Primary tools

Blender for asset inspection and cleanup. A-Frame for the eventual submitted scene.

What counts today

Finding and evaluating strong room assets, not building a polished VR scene.

What to ignore today

Any older Track 3 instructions that require Unity scenes, Unreal starter repos, or headset setup.

What Students Should Do Today

  1. Install Blender and confirm it opens.
  2. Download at least three candidate room models in `.glb`, `.fbx`, or `.obj`.
  3. Import each candidate into Blender and inspect scale, hierarchy, openings, and materials.
  4. Create one short room sheet per candidate explaining why it is or is not a good benchmark room.
  5. As a team, shortlist the strongest rooms for the shared class corpus.

What To Submit Today

Deliverable 1: Team charter

One page stating the team’s goal, initial roles, naming conventions, what counts as a stable artifact, and how blocked work will be handled.

Deliverable 2: Benchmark room shortlist

A small set of candidate rooms, each with a room sheet stating source, format, condition, likely supported transformations, and likely breakage risks.

How To Judge A Room

  • A good room has realistic scale. A door should look like a door, not a tower.
  • A good room has inspectable hierarchy. Walls, floors, ceilings, and openings should not all be fused into one useless mesh.
  • A good room supports controlled edits. You should be able to imagine changing ceiling height, enclosure, windows, light, or materials without destroying everything else.
  • A good room is portable. Engine-neutral formats are preferable because Track 3 should not depend on one fragile engine path.

Where To Get Assets

Sketchfab

Use downloadable interior or room models. Prefer `.glb` when available.

TurboSquid

Use as a secondary source when the model hierarchy is clean enough to survive editing.

Poly Haven

Best for HDRIs, materials, and textures. Useful even when the room shell comes from somewhere else.

Meshy / Tripo

Use these for isolated props or experimental room candidates, not as the only benchmark corpus source.

Minimal Folder Structure

Students do not need a complex engine project today. This is enough:

mkdir -p track3_workspace/{benchmark_rooms,generated_assets,variants,logs} cd track3_workspace

What Comes Next

After today, the sequence should be:

  1. Week 2: define factors and operator specifications.
  2. Week 3: produce one modest end-to-end transformation proof.
  3. Week 4–5: build reliable variants and package them cleanly.
  4. Later: connect that work to the A-Frame workbook and the final instrumented scene.

The right academic structure is: benchmark rooms first, then controlled transformations, then one experimentally legible scene.
