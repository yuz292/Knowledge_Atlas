        Access model
        Who sees what — canonical reference for the site.

        Only 160 Student and Instructor require a login. Every other role is a presentation mode: a visitor to the site picks one, and the navigation, landing page, and content emphasis adjust accordingly. An admin can slip into any of these roles without losing admin privileges; a small amber banner at the top of every page confirms the impersonation and offers a one-click return.

        Integration — for every other page on the site
        Include the companion script so pages respect the same rules.

        Add one line inside every page's <head>:

        <script src="ka_user_type.js" defer></script>

        That script renders the impersonation banner (when applicable), exposes window.KA.userType to page code, and guards any element marked data-ka-requires="160-student" by hiding it from unauthenticated visitors while admin-impersonators pass through transparently.
