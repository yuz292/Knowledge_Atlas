Question for the week: What do you now have that is genuinely usable, and what have you learned about the hard parts?

Use AI well

Before you ask AI for help, make sure you can state what you are trying to learn this week and what evidence would count as a useful answer.

Class meetings

Final integration meeting
Start here: Pretend another lab receives your package tomorrow. What can they actually use?

In class

  • Run the final demo as a studio, not as isolated teams.
  • Show one complete path from room choice to transformed variant to packaged output.
  • Make limitations explicit: unsupported requests, fragile operators, asset classes that remain difficult, and manual steps that still matter.
  • Discuss what would be the next three steps if the project continued for another month.

Week deliverables

D11 Final studio package

Rationale
The final package should show that the class built more than disconnected experiments. It should present a coherent workflow and its current limits.

What to submit
One final bundle containing benchmark room reference, implemented operators, translation pack, workflow prototype, one or more variants, logs, and a short reflection.

What good work looks like
The package is coherent, honest, and portable. It lets another person understand what the class achieved without needing to sit in the room.

Example
A folder or repo with a quick-start document, benchmark-room list, operator evidence, worked request examples, before and after captures, and a clear note on unsupported features.
