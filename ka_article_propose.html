Step 1A — Submit Citations
Paste one citation per line or upload a citation file. This supports Track 2 workflows where you screen abstracts first and only fetch PDFs later.
Accepted file types for citation intake: .txt, .csv, .ris, and .bib. In the real backend these would be parsed by the AF importer before de-duplication and staging.
Step 1B — Upload PDFs
Drop one PDF or a whole batch. The system extracts metadata, checks for duplicates immediately, and stages non-duplicates for later review. For Track 2, a PDF without at least author + title or a pasted citation should be treated as incomplete.
Immediate gate: duplicate detection happens as soon as the PDF is inspected. If KA already has the article, the upload is stopped and no second permanent copy is kept.
Deferred gate: relevance is reviewed asynchronously in the nightly intake process. That means contributors can move quickly, while maintainers make the harder inclusion decisions in batch.
Track 2 rule: do not stage an orphan PDF. Before staging, include either a pasted APA citation or enough metadata to reconstruct one quickly: first author + title, or DOI + title. That makes abstract recovery, duplicate detection, and later repair materially easier.
Security gate: uploaded files are untrusted until validated. The real intake backend must verify file type by content, quarantine uploads, hash them, reject encrypted or malformed files, and process them in an isolated worker before any promotion into the main corpus.
Storage rule: rejected staged PDFs should be deleted promptly after review to save space. Metadata and rejection reason remain in the audit trail.

Step 2 — Inspect Batch Results
Each intake item gets a first-pass metadata extraction. Duplicate matches are blocked immediately. Public submissions may remain provisional, but Track 2 submissions should not be staged until they have a minimally usable citation record.

Step 3 — Optional Citation Cleanup
If automatic resolution is weak, refine the record before staging. For Track 2, this is not optional: every staged item should have enough citation information to recover an abstract and create a stable APA record, and the abstract itself should be included whenever your search tool already exposed it.
Selected file metadata
In the real pipeline this step would call Crossref / OpenAlex / Semantic Scholar. In this demo it uses DOI or title matching against a small mocked resolver. Including an author string materially improves citation recovery quality.
Step 4 — Nightly Review Policy
What happens after staging.
  • Duplicates: rejected immediately; no permanent second copy stored.
  • Staged pending review: held for the nightly reviewer pass.
  • Accepted: moved into the extraction queue.
  • Rejected as irrelevant: PDF deleted promptly after review; metadata and decision retained.
