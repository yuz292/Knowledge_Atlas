Knowledge Atlas agents

Structured roles for writing, charting, audit, and design

A K-ATLAS agent is a structured role with a citation-grounded reference group, a one-sentence mission, a required-output pack, and a checklist of things it must never do. Agents are meant to be run as sub-prompts by you — whether as a student preparing a Track 4 deliverable, a contributor writing a page's copy, or a reviewer auditing a design proposal. The full agent source lives under /agents/ in the repo; this page is the human-readable index.

What this page is — and what it isn't

This page is a directory, not an execution surface. Agents are prompt contracts — named roles, reference groups, required outputs — that run in an LLM session on the backend (Claude, GPT, Gemini, or a local model). There is no "Run this agent" button on this page. The agents work by being read into a sub-prompt, not by being clicked.

How to invoke an agent

  1. Open the agent's source (/agents/*.md) to review its reference group, required inputs, and required outputs.
  2. Copy the "Prompt to run this agent" block at the bottom of the source.
  3. Paste into a Claude / GPT / Gemini session — Cowork, Claude Code, the web client, or a backend service with an API key — with the specific page, chart, or draft attached as context.
  4. The LLM returns the structured report the agent specifies. Apply the findings; re-run the agent on the revised draft if you want a second pass.

A browser-invocation surface (a "Run agent" button that posts the page's current content to a backend LLM and streams back the structured report) is a plausible future feature — it would need a backend endpoint with LLM credentials, a task queue, and a submission UI. It is explicitly not in scope today. For now, the agents run where every other LLM sub-prompt runs: in your own LLM session.

Agent · Science-Writing

K-ATLAS Science-Writing Agent

Writes page copy, microcopy, explanations, warnings, example questions, and labels. Optimises for clarity, conceptual accuracy, and epistemic honesty — never hides uncertainty, never flattens expert pages into public prose.
Reference group: Pinker (classic style, curse of knowledge), Williams (Given-New, stress position), Lanham (Paramedic Method), Sword (zombie nouns), Sacks, Sagan (scale bridging), Yong (slow complexity), Gawande, Carson, Doumont (structure as communication).
Required outputs per page: 13 items — one-sentence purpose, short intro, expanded explainer, section labels, CTA wording, example questions, warnings, empty/loading/error states, tooltips, beginner variant, expert variant.

Agent · Science-Graphics

K-ATLAS Science-Graphics Agent

Audits, critiques, and directs the design of any data visualisation on the site. Optimises for signal-per-pixel, perceptual accuracy, uncertainty visibility, and familiar chart vocabulary. Runs before a chart is rendered, or as a critique agent after.
Reference group: Tufte (data-ink, small multiples), Cleveland & McGill (perceptual-accuracy ordering), Bertin (retinal-variable budget), Christiansen & Montanez (familiar vocabulary first), Cox (iterate chart form), FT Visual Vocabulary (chart-to-task mapping), Krzywinski & Altman (Points of View), PLOS Comp Bio "Ten simple rules".
Required outputs: 10 items — message check, chart-form fit, data-ink audit, encoding audit, uncertainty check, caption check, axis/scale check, colour audit, anti-eye-candy check, prioritised redesign list.

Agent · Usability Critic

K-ATLAS Usability Critic Agent

Conducts structured heuristic evaluation of any page or visualisation. Produces a verdict table and prioritised repair list — one paragraph per violated criterion. Does not soften verdicts; does not skip criteria because the page "mostly works."
Reference group: Nielsen & Molich (10 heuristics), Shneiderman (8 golden rules), plus 17 info-design criteria. 35 dimensions total.
Required outputs: per-criterion finding, severity rating (cosmetic / minor / major / catastrophe), verdict table, prioritised repair list. Integrates with the in-browser ka_usability_critic.js panel students can launch on any live page.

Agent · GUI Design

K-ATLAS GUI Agent (v3)

Designs page layout, interaction patterns, and progressive-disclosure structure. Runs after the Science-Writing Agent has produced the copy pack, so that layout decisions are made around language that is already clear.
Reference group: Norman (system image / user model), Krug (don't make me think), Nielsen & Loranger (prioritising web usability), Rosenfeld & Morville (information architecture), Hearst (search interfaces), Wroblewski (form design).
Required outputs: page layout sketch, interaction pattern per affordance, progressive-disclosure tier structure, responsive breakpoint behaviour, accessibility notes.

Agent · Experimental Evaluate

K-ATLAS Experimental-Evaluate Agent

Audits experimental designs for methodological soundness before data collection — pre-registration checklist, power analysis, DV/IV clarity, confound audit, replication plan. Catches the common "what would falsify this?" failure at the design stage rather than the peer-review stage.
Reference group: Chambers & Munafò (registered reports), Simmons/Nelson/Simonsohn (false-positive psychology), Cumming (the new statistics), Munafò et al. (manifesto for reproducible science).
Required outputs: pre-registration checklist pass/fail, power estimate with assumptions, confound register, falsification condition, replication plan.

When to chain agents

The five agents compose cleanly into a workflow. For a new page with data visualisations:

  1. Science-Writing Agent → produce the copy pack (title, caption, explanatory paragraphs, tooltips, glossary entries).
  2. Science-Graphics Agent → decide the chart form and the encoding for each data panel; draft captions that guide attention.
  3. GUI Agent v3 → place the copy and charts in a layout; decide progressive-disclosure tiers; name the affordances.
  4. Usability Critic Agent → audit the assembled page against Nielsen + Shneiderman + info-design criteria; produce a prioritised repair list.
  5. Experimental-Evaluate Agent (if the page publishes empirical claims) → audit the methodology of the claims the page is reporting.

Track 4 students run a version of this chain on their T4.a / T4.c / T4.f deliverables. Contributors writing K-ATLAS journey-page content should run at minimum steps 1 and 2 before pushing a draft.
